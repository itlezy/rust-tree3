# SEO and Discovery Checklist

This repository is a technical Rust project, so discovery depends mostly on GitHub search, package metadata, README clarity, and inbound links from math/programming communities.

## Primary Search Intent

Target people searching for:

- TREE(3) visualization
- TREE(3) sequence explorer
- Rust TREE(3) implementation
- Kruskal tree theorem implementation
- homeomorphic tree embedding algorithm
- rooted labeled tree SVG generator
- googology TREE function

## Repository Metadata

Recommended GitHub description:

> Rust CLI for exploring Harvey Friedman's TREE(3), Kruskal tree theorem sequences, homeomorphic tree embeddings, and SVG visualizations.

## README Checklist

- The H1 includes the main term `TREE(3)` plus supporting terms such as Rust, Kruskal trees, googology, and SVG visualization.
- The first paragraph explains the project in plain search-friendly language.
- The README includes examples of generated trees near the top, so visitors understand the output immediately.
- The quickstart command appears before the deeper mathematical and implementation details.
- The references section links to authoritative background material.

## External Discovery

Useful places to share the project after publishing:

- GitHub topic pages listed above
- Rust community project threads
- Math, combinatorics, graph theory, and googology forums
- A short article explaining how the embedding algorithm works, linking back to the repository
- A gallery page or GitHub Pages site that embeds `docs/examples/overview.svg`

## Optional Next Step

If the project is published as a crate, add a `license` or `license-file` field to `Cargo.toml` and confirm that the crate page uses the same description, keywords, and repository link.
